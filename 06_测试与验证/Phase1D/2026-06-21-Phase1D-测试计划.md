# Phase 1D 测试计划

日期：2026-06-21
阶段：Phase 1D
定位：Alpha 前内部测试 / 小范围测试准备
项目：轻选 Agent / 2代电商agent

## 1. 阶段结论

Phase 1C 已完成流程闭环和远端备份。Phase 1D 不新增功能，重点验证：

- 页面是否稳定
- 用户是否能跑通流程
- AI 输出是否有参考价值
- 风险提示是否足够保守
- 任务记录是否可追溯
- 人工复核是否明确

当前不得宣称商业化完成，不得宣称全自动化完成。

## 2. 测试边界

本阶段允许：

- 页面访问测试
- 表单可用性测试
- 历史任务查看测试
- 截图归档
- 文档记录
- 少量人工控制下的商品链路测试

本阶段禁止：

- 新增功能
- 自动发布
- 自动投广告
- 自动爬虫
- 自动外部平台操作
- 数据库结构变更
- 无确认地调用真实 AI
- 把 AI 模拟测试写成真人 Alpha

## 3. 推荐测试流程

每个商品建议按以下流程测试：

1. 素材输入或商品描述整理
2. 货源判断 `/sourcing`
3. 风险排查 `/risk`
4. 小白结论 `/summary`
5. Listing / 关键词 / 利润初步判断
6. 任务中心 `/tasks` 查看记录
7. 人工复核
8. 记录问题和改进建议

## 4. 推荐测试商品

建议优先选 5–8 个商品，覆盖低风险、中风险、高风险：

低风险：

- 宠物慢食碗
- 桌面手机支架
- 折叠露营桌

中风险：

- 宠物饮水机
- 硅胶折叠水杯
- 磁吸收纳配件

高风险：

- 儿童电动牙刷
- 婴儿用品
- 食品接触类容器
- 带电池产品

## 5. 成功标准

Phase 1D 通过标准：

- 核心页面无 500 / 404
- lint 通过
- build 通过
- 主要流程能跑通
- 高风险商品不会被轻易推荐
- 任务记录能追溯
- 测试问题有记录
- 输出文案不夸大商业化状态

## 6. 不通过标准

出现以下情况，不应进入下一阶段：

- 关键页面无法访问
- 高风险商品被推荐为新手可做
- AI 输出明显误导但无人工复核提醒
- 任务记录丢失或打不开
- 访问密码失效导致核心流程不可用
- 测试报告缺失
