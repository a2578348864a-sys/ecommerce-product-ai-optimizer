## 1. V2 不做什么

以下边界永久锁定，V2/V3/V4 均不突破：

- 不做全自动采购
- 不做自动发布商品
- 不做自动投广告
- 不做外部平台自动操作（登录、下单、联系供应商、改库存/价格、处理订单/退款）
- 不承诺 AI 结果一定正确
- 不在用户未确认的情况下对外执行任何资金/账号/客户操作

## 2. V2 MVP 做什么

从"多个独立分析页面各自跑"升级为"一条工作流串起来"：

### 2.1 入口
- 用户在 `/opportunities` 机会雷达中看到候选品排行榜
- 选择 1 个候选商品 → 进入 V2 工作流（`/v2-lab` 或 `/workflow`）

### 2.2 工作流步骤

| 步骤 | 名称 | 说明 | 复用现有能力 |
|------|------|------|-------------|
| 1 | 货源确认 | 1688 搜索词、MOQ、价格带、供应商数量 | `/api/agents/sourcing` |
| 2 | 风险确认 | 认证、侵权、物流、售后、平台规则 | `/api/agents/risk` |
| 3 | 利润粗算 | 售价、成本、运费、佣金、毛利率 | `/products/new` 利润模块 |
| 4 | 合规确认 | 目标市场认证要求、平台资质要求 | `/api/agents/summary`（合规部分） |
| 5 | 新手适合度 | 综合判断：新手能不能做，难度在哪 | `/api/agents/summary`（新手结论） |
| 6 | 下一步动作 | 建议人工做什么：找供应商、索样、测品、查认证 | aggregator 聚合输出 |

### 2.3 人工确认机制
- 每个步骤的结果旁边有明确的「人工确认」按钮/标记
- 6 步全部确认后，生成最终「小白结论」和「下一步清单」
- 保存到 `/tasks`，类型 `workflow`，附带 decisionStatus

### 2.4 不自动执行
- 所有「下一步动作」只是建议，不是自动化操作
- 不自动联系供应商
- 不自动下单
- 不自动上架
- 不自动对外发送任何内容

## 3. V2 页面草图

### 3.1 新页面：`/v2-lab`

页面结构（三栏 + 底部）：

```
┌─────────────────────────────────────────────────────┐
│  ⚠️ V2 工作流沙盒 — 原型阶段，不是已上线功能          │
├────────────┬────────────────────┬───────────────────┤
│            │                    │                   │
│  候选商品   │   工作流步骤        │  人工确认清单      │
│            │                    │                   │
│  · 商品 A  │  1. 货源确认 ✓     │  □ 货源已确认      │
│  · 商品 B  │  2. 风险确认 →     │  □ 风险已确认      │
│  · 商品 C  │  3. 利润粗算       │  □ 利润已粗算      │
│            │  4. 合规确认       │  □ 合规已确认      │
│            │  5. 新手适合度     │  □ 适合度已判断    │
│            │  6. 下一步动作     │  □ 动作已确认      │
│            │                    │                   │
├────────────┴────────────────────┴───────────────────┤
│  💡 下一步建议：建议先从商品 A 开始，优先确认货源和利润  │
└─────────────────────────────────────────────────────┘
```

### 3.2 页面规格
- 纯静态 mock，不调用 API，不调用 AI，不写数据库
- 所有数据为硬编码演示数据
- 页面顶部明确标注「V2 工作流沙盒」
- 不加入主导航（WorkspaceSidebar），仅通过 URL 直接访问
- 仅供内部规划和截图讨论

## 4. V2 成功标准

- [ ] 用户能看懂下一步该干嘛
- [ ] 用户知道哪些步骤需要人工确认
- [ ] 用户不会误以为系统能自动赚钱
- [ ] 系统不扩大真实 AI 成本（V2 初期 0 AI 调用）
- [ ] 不影响 V1.5 生产稳定版
- [ ] `/v2-lab` 不与现有页面冲突
- [ ] mock 原型获得用户认可后再进入真实开发

## 5. 与现有功能的关系

| 现有功能 | V2 关系 |
|---------|--------|
| `/opportunities` | V2 入口，候选品来源 |
| `/sourcing` | 独立使用仍保留；V2 中复用 AI 能力 |
| `/risk` | 同上 |
| `/products/new` | 利润计算模块可复用 |
| `/summary` | 小白结论改为 V2 流程中的最终步骤 |
| `/tasks` | 新增 `workflow` 类型任务 |
| `/viral` | V2 暂不涉及，独立保留 |
| `/materials` | V2 暂不涉及，独立保留 |

**核心原则：不废弃任何现有页面，V2 为增量叠加。**

## 6. 实施阶段（草案）

### Phase 0（当前）：沙盒规划 + mock 原型
- 本文档
- `/v2-lab` 静态 mock 页面
- 不接 API、不接 AI、不写 DB
- 在 `feature/v2-workflow-sandbox` 分支开发

### Phase 1：/v2-lab 最小可用（待评审后）
- 接 `/api/opportunities` 获取候选品列表
- 选中商品后串行调用 sourcing → risk → summary
- 结果展示在 6 步工作流中
- 人工确认后保存到 `/api/tasks`（type=workflow）

### Phase 2：工作流增强（待定）
- 并行/条件跳过步骤
- 利润粗算模块
- 下一步动作聚合
- 从 `/opportunities` 直接跳转 `/v2-lab?product=xxx`

### Phase 3：正式发布（待定）
- 页面改名 `/workflow`
- 加入主导航
- 生产部署

## 7. 风险与约束

- V2 不能导致 AI 调用次数激增（单商品 3 次 vs 机会雷达批量 30×3）
- 工作流步骤顺序需保持灵活性（可跳过、可重试）
- mock 原型不能给用户造成"功能已可用"的错觉
- 沙盒分支不 merge 前需经过生产 review
