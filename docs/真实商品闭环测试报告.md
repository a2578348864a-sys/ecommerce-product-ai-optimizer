## 2. 总结论

结论：需先修关键问题，暂不建议给真实跨境电商小白内测。

核心原因：

1. `/tasks` 列表能搜索、筛选、展示记录，但点击“查看详情”后详情页显示“任务记录不存在或已删除”，任务详情闭环不可用。
2. `/sourcing` 和 `/risk` 生成成功，但页面没有保存到任务中心入口，导致 `/summary` 只能聚合选品体检、爆款拆解、素材接收 3 类结果，无法聚合货源和风险。
3. `/products/new` 英文上架文案会恢复上一商品的本地文案，换商品后先看到旧商品文案，存在跨商品串货风险；必须手动重新生成才正确。
4. 访问密码不是跨页面稳定复用，`/summary` 每个商品都需要用户重新手动输入，真实小白流程会卡顿。
5. 任务中心仍显示“删除”按钮，本轮未点击，但对测试/内测用户存在误删风险。

## 6. 最大问题

### P0

1. `/tasks` 详情页不可用：列表中点击“查看详情”后显示“任务记录不存在或已删除”。这会直接破坏“保存 -> 查看 -> 复核”的闭环。
2. `/sourcing`、`/risk` 不能保存到任务中心：两个关键 Agent 结果无法沉淀，`/summary` 只能看到 3 类结果。
3. `/products/new` 英文上架文案跨商品缓存串货：新商品页面会先显示上一商品的英文文案，真实小白可能误复制错误内容。

### P1

1. `/summary` 对高风险商品权重不足：儿童安全用品风险页判高风险，但小白结论仍给“可以做”。
2. 访问密码跨页面状态不稳定：`/summary` 每个商品都需要重新手动输入，用户体验差。
3. `/tasks` 搜索配合类型筛选时只能看到部分记录，容易让用户误以为记录丢失。
4. `/sourcing` 个别风险卡片出现“待确认 / 未提供详细说明”的空泛格式。

### P2

1. 移动端无页面级横向滚动，但顶部工作流导航为横向滚动区，部分链接默认在视口外。
2. `/materials` 对通用素材提取偏泛，缺字段补充建议还不够强。
3. 本地测试环境曾出现 `next start` 页面 500、dev 进程被中断等环境稳定性问题；最终使用 `npm.cmd run dev -- -p 3005` 完成测试。

## 7. 下一轮最该修的 5 点

1. 修 `/tasks/[id]` 详情页读取逻辑，确保从列表点击任意任务都能打开完整详情。
2. 给 `/sourcing` 和 `/risk` 增加“保存到任务中心”，并在 `/summary` 聚合这两类任务。
3. 修 `/products/new` 英文文案缓存策略：按商品名/平台隔离，商品变化时不自动展示旧商品文案。
4. 调整 `/summary` 决策权重：只要风险结果为高风险，默认至少降为“谨慎做”；儿童/食品接触类目不得轻易给“可以做”。
5. 统一访问密码状态，避免 `/summary` 每个商品都要重新输入；同时不要把密码写入报告或日志。

## 9. 最终判断

现在不能给真实跨境电商小白试用。

还差几个关键点：

1. 任务详情页必须能打开，否则用户无法复核历史结果。
2. 货源和风险必须保存并进入小白结论，否则“闭环”不完整。
3. 跨商品文案串货必须修，否则容易把 A 商品文案复制到 B 商品。
4. 高风险品类的结论必须更保守，尤其儿童用品、食品接触、宠物用品。
5. 密码和页面状态要更稳定，减少小白反复输入和误操作。

本轮建议：先修 P0，再做第二轮真实商品闭环复测。
