## 5. 对 P1 的处理判断

### P1-1：食品接触类 green 判定需收紧

- **当前不阻塞收口**。这是一个硬规则收紧，需要改代码。
- 建议在 Phase 1E 处理：对食品接触/儿童/带电/磁铁四类商品统一收紧 green 判定，至少 yellow 起步。
- 当前 Alpha 演示中可以口头提醒用户注意食品接触类风险。

### P1-2：/tasks 决策状态全 pending

- **当前不建议直接改数据库伪造决策状态。**
- "pending"代表"等待人工复核"——这本身是 Alpha 阶段的真实状态，不是 bug。
- 不建议为了让列表"好看"而制造假数据。
- 后续 Phase 1E 可做正式的"人工决策"功能：允许用户标记 continue/need_more_info/eliminated，并记录决策时间和理由。

### P1-3：同商品风险等级不一致

- 根本原因是同商品在不同时间、不同输入下得到不同 risk 分析结果。
- 这不完全是 bug——同一个商品可以多次分析，结果可能因输入侧重点不同而有差异。
- 建议后续统一 risk/summary/product 的风险口径，增加"基于最新分析覆盖旧结果"的提示。
